# Master Project Workflow: Uber Clone

This document provides a high-level visual map of how the different modules of the system interact to provide a complete ride-hailing experience.
---

## The End-to-End Lifecycle

The platform operates through four primary"Experience Loops":

### 1. The Onboarding Loop
How users and drivers enter the ecosystem.
- **Rider**: Signup → Welcome Promo → Profile Setup.
- **Driver**: Signup → Document Upload → Admin Verification → Activation.

### 2. The Booking Loop (The Core)
The real-time matching experience.
- **Request**: Rider selects destination → Fare calculation.
- **Matching**: Redis GEO Search → Celery Task → Driver Offer (WS).
- **Handshake**: Driver Accepts → OTP Generated → Driver En Route.

### 3. The Transport Loop
Live tracking and safety.
- **Verification**: OTP check at pickup.
- **Tracking**: GPS streams (POST) → Snapping Logic → Rider Map Update (WS).
- **Safety**: SOS Button → Real-time Admin Alert.

### 4. The Terminal Loop
Finalizing the transaction.
- **Completion**: Driver arrives → Final distance calc → Fare commit.
- **Payment**: Razorpay Capture / Wallet Debit → Triple-Entry Ledger.
- **Feedback**: Star rating → Driver trust score update.
