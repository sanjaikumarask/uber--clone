# Master Project Workflow: Uber Clone

This document provides a high-level visual map of how the different modules of the system interact to provide a complete ride-hailing experience.
---

## The End-to-End Lifecycle

The platform operates through four primary"Experience Loops":

### 1. The Onboarding Loop
How users and drivers enter the ecosystem.
- **Rider**: Signup → Welcome Promo → Profile Setup.
- **Driver**: Signup → Document Upload → Admin Verification → Activation.

### 2. The Booking Loop (The Core)
The real-time matching experience.
- **Request**: Rider selects destination → Fare estimate → Ride created (`SEARCHING`).
- **Matching**: `transaction.on_commit` → `find_driver_and_offer_ride()` → Redis GEO Search → Unique Driver Offer via WebSocket.
- **Handshake**: Driver Accepts → OTP Generated → Driver En Route (`ASSIGNED` or `OFFERED`).

### 3. The Transport Loop
Live tracking and safety.
- **Verification**: OTP check at pickup (`VerifyOtpView`).
- **Tracking**: GPS streams via **WebSocket** (`ws/tracking/driver-location/`) → Road Snapping (every 10th ping) → Triple Broadcast (Rider, Driver, Admin).
- **Safety**: SOS Button → Persistence in `Emergency` model + Real-time Alerts.

### 4. The Terminal Loop
Finalizing the transaction.
- **Completion**: Driver arrives → Final distance calc → Ledger Settlement (`settle_driver_payout`).
- **Payment**: Razorpay Capture / Wallet Debit → Triple-Entry Ledger (`LedgerEntry`).
- **Feedback**: Star rating → Automated Driver Trust Score recalculation.
- **Cleanup**: terminal state committed → `SequenceFencer.clear_fence`.
